Teste de Conexao Supabase

RDO Auto — validacao de credenciais

    Instrucoes

    1. Substitua a URL acima pela URL do seu projeto Supabase (ex.: https://abc123.supabase.co).

    2. Crie a tabela rdo_teste no SQL Editor do Supabase:

    -- Execute este comando no SQL Editor do Supabase
CREATE TABLE IF NOT EXISTS rdo_teste (
  id          BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  data        DATE NOT NULL DEFAULT CURRENT_DATE,
  profundidade NUMERIC(6,2),
  descricao   TEXT,
  supervisor  TEXT,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

    3. Desabilite RLS (apenas para este teste):

    -- No SQL Editor, execute:
ALTER TABLE rdo_teste DISABLE ROW LEVEL SECURITY;

    Ou, se preferir manter RLS ativo, crie uma politica temporaria:

    -- Politica que permite INSERT e SELECT para qualquer um
-- (apenas para teste; remova depois)
CREATE POLICY "teste_rdo_public"
ON rdo_teste
FOR ALL
USING (true)
WITH CHECK (true);

    4. Clique em "Testar Conexao". O teste insere um registro e depois consulta os ultimos 5 registros da tabela.

    5. Se tudo funcionar, voce vera uma lista com os registros. Caso contrario, a mensagem de erro mostrara o codigo HTTP e detalhes para diagnostico.
