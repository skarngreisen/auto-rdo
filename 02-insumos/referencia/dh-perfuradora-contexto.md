## 2. Diagnóstico Consolidado

### 2.1 O Problema Central

**Amadorismo empresarial.** A empresa não mede quase nada, as gerências não remam na mesma direção, e todo mundo apaga incêndio o tempo todo — sem tempo para planejar ou registrar.

### 2.2 Os Gargalos por Voz dos Stakeholders

O Murilo relatou que "a névoa está se dissipando, está clareando" — as áreas estão começando a se organizar. A Cintia já provou que é possível medir, com uma planilha de 13 abas construída RDO a RDO. Esta consultoria não começa do zero: ela **acelera o que já está em movimento** e libera as pessoas para usarem os dados, não apenas coletá-los.

| Causa                                                                  | Horas  | % do Downtime | Controlável? |
| ---------------------------------------------------------------------- | ------ | ------------- | ------------ |
| PARC DAS ARTES (cliente) — tubos de revestimento (8d) + decisões Saerp | 9d 17h | **43%**       | Parcial      |
| Falta de Equipe / DTS / Mão de Obra                                    | ~4d    | **18%**       | ✅ SIM        |
| Manutenção Corretiva de Equipamentos                                   | ~4d    | **18%**       | ✅ SIM        |
| Atividades-Meio / Logística / Administrativo                           | ~2d    | **9%**        | ✅ SIM        |
| Materiais / Insumos internos                                           | ~2d    | **9%**        | ✅ SIM        |
| Chuva                                                                  | ~14h   | 3%            | Não          |

> **Falta de Equipe + Manutenção + Materiais internos + Logística = 54% do horas paradas** → coisas que a DH pode controlar diretamente. O restante (46%) depende de alinhamento contratual com o cliente.

#### NPT vs. Downtime — O Mecanismo

```
Manobras são inerentes à perfuração (NPT = não produtivo, mas aceitável)
  → Cada poço tem um BASELINE de manobras esperadas
    → Falta de equipe força mais manobras que o previsto
      → NPT vira DOWNTIME (hora perdida evitável)
        → Sem linha de base calculado, o desvio fica INVISÍVEL
```

**Projeto Parc das Artes:** deveria ter ~5 manobras no cronograma. Fizeram mais de 90.

### 2.3 A Síndrome Organizacional

> **"Geólogo é gestor, não só geólogo."** — Halleph

O cargo diz "geólogo", a função real é "gestor de obra": segurança, logística, suprimentos, financeiro, RH — tudo recai sobre eles. Isso gera uma **nebulosidade** onde todo mundo faz de tudo, ninguém sabe onde termina sua responsabilidade, e o incêndio vira o normal.

## 3. Framework: OKRs + CFRs

### 3.1 Por que OKRs?

- A empresa **não mede quase nada** — OKRs forçam a começar a medir com disciplina
- Ajuda a ajustar prioridades e **alinhar as gerências** com a visão do diretor
- Estrutura colaborativa (bottom-up + top-down) para engajar as equipes

### 3.2 Cuidados na Aplicação

- OKRs podem soar "kum ba yah" ou "Silicon Valley" para profissionais seniores
- A empresa ainda não tem maturidade para KPIs — pode gastar tempo medindo coisas sem aproveitar
- KR é métrica, mas difere de KPI no imaginário corporativo — KPI remete a bônus e meta, KR não
- **Traduzir para o público operacional:** "OKR é só um jeito de escolher 2-3 prioridades por mês e todo mundo saber o que importa"

### 3.3 Plano de Aplicação

1. **Início do contrato:** Treinamento de OKRs para diretor + 3 gerentes
2. **Durante:** Criação dos primeiros OKRs + ritos de CFR (Conversas, Feedback, Reconhecimento)
3. **Final:** Fechamento dos OKRs — medir o que foi alcançado, o que não foi, e deixar a máquina rodando

### 3.4 CFRs (Conversations, Feedback, Recognition)

- Conversas semanais de 15 min entre gerente e supervisor para revisar o que funcionou/não funcionou
- Feedback loop: fazer → medir → aprender → ajustar
- Reconhecimento como ferramenta de engajamento

## 4. Os 3 Produtos da Consultoria

### 4.1 Produto 1 — Radar Operacional (Mês 1-2)

**O que é:** Um painel que unifica dados de campo em tempo real, comparando o real vs. o linha de base previsto.

**Componentes:**

| Componente                                          | O que faz                                                                                                                | Quando |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ------ |
| **Coleta de dados via WhatsApp (RDO Automatizado)** | Bot pergunta as questões padrão do RDO, supervisor responde em áudio/texto. Zero instalação, 2-3 min/dia                 | Mês 1  |
| **Modelo de Linha de Base por Projeto**             | Antes de cada obra, supervisor preenche modelo (30 min) com tipo de poço, equipe, manobras esperadas. Gera linha de base | Mês 1  |
| **Painel (Power BI / Looker)**                      | Compara real vs. linha de base, alertas de desvio de NPT, horas paradas, produtividade                                   | Mês 2  |

**O que NÃO é:**

- ❌ Não é MS Project com alocação de recursos
- ❌ Não exige PMO dedicado
- ❌ Não exige atualização semanal de cronograma
- ❌ Não depende de app novo (usa WhatsApp)

**O que resolve:**

- ✅ Dá visibilidade de onde estão os gargalos operacionais
- ✅ Cria o linha de base de manobras que hoje não existe
- ✅ Libera os geólogos do trabalho braçal de RDO
- ✅ Dá segurança para o Murilo e outros decidirem com dados objetivos
- ✅ Protege a receita: RDO atrasado = risco de R$400-500k em pleitos

**Restrições de design:**

- Baixa complexidade de implementação
- Baixo custo operacional corrente (sem consumo massivo de tokens)
- Não pode exigir que o supervisor pare por mais de 2-3 minutos
- Precisa funcionar em área com internet limitada (campo)

---

### 4.2 Produto 2 — OKRs + Acompanhamento de CFRs (Mês 1-3)

**O que é:** Programa de gestão estratégica usando OKRs como sistema de execução, com ritos de CFR semanais.

| Etapa        | Atividade                                               |
| ------------ | ------------------------------------------------------- |
| **Semana 1** | Treinamento de OKRs para diretoria + gerentes           |
| **Mensal**   | Criação e revisão de OKRs (2-3 KRs por objetivo)        |
| **Semanal**  | Ritos de CFR mediados pelo consultor                    |
| **Mês 3**    | Fechamento dos OKRs — retrospectiva e lições aprendidas |

**Por que OKRs agora:**

- Força a disciplina de medição que a empresa não tem
- Quebra o ciclo de "apagar incêndio" ao forçar escolhas difíceis
- Prepara a empresa para o nível de governança que a Vale exige

---

### 4.3 Produto 3 — Melhoria Contínua + Facilitação de Ritos (Mês 2-3)

**O que é:** Treinamento e facilitação de práticas de melhoria contínua, sem criar POPs (já há alguém contratado para isso).

| Atividade                            | Detalhamento                                                                                      |
| ------------------------------------ | ------------------------------------------------------------------------------------------------- |
| **Treinamento de melhoria contínua** | Feedback loops, RECS (Retorno de Experiência), ciclo planejar-executar-revisar-ajustar            |
| **Instituição de ritos**             | Reuniões semanais de 15 min, DDS com componente operacional, listas de verificação pré-perfuração |
| **Mediação das primeiras reuniões**  | Consultor senta junto, facilita, e vai tirando o cabresto — "aprender fazendo"                    |
| **Ferramenta simples de RECS**       | Local para registrar lições aprendidas (Google Forms, canal, ou repositório)                      |

**O que NÃO é:**

- ❌ Não é criação de POPs (já existe consultor externo para isso)
- ❌ Não é documentação pesada ao estilo F4 (1.500 páginas de capturas de tela)

**O que É:**

- ✅ É ensinar o time a **usar** os POPs que estão sendo criados
- ✅ É criar o hábito de registrar lições aprendidas
- ✅ É mostrar como aplicar melhoria contínua no dia a dia de campo
